# wireframe

## 背景

本文件承接：

## 线框原则

1. 2FA 是独立账号安全能力，位于 `账号设置 / 安全中心`。
2. 支付密码、提现、充值页面只消费安全服务策略。
3. 页面不写死风险规则，只展示策略返回后的状态。
4. 支付密码设置 / 重置使用合并表单，不再固定要求邮箱验证码。
5. 资金确认页使用合并表单，支付密码和两步验证同屏输入。
6. 命中 2FA 但未初始化或锁定时，强阻断，不自动回跳原操作。

## W-01 安全中心两步验证安全项

### 内容

- 安全中心页面标题。
- 两步验证安全项。
- 当前状态：未启用、已启用、已锁定、不可用。
- 验证方式：手机短信或验证器动态码。
- 最近启用时间。
- 操作按钮：启用两步验证、查看找回方式、联系客服。

### 状态

| 状态 | 展示 | 操作 |
|---|---|---|
| 未启用 | 「两步验证：未启用」 | 启用两步验证 |
| 已启用 | 「已启用 · 手机短信」或「已启用 · 验证器动态码」 | 查看找回方式 |
| 已锁定 | 锁定原因、剩余时间 | 查看找回方式 / 联系客服 |
| 不可用 | 通道不可用或缺少绑定 | 去绑定手机号 / 稍后重试 / 联系客服 |

## W-02 大陆账号启用手机短信验证

### 结构

```text
启用手机短信验证
当前手机号：+86 138****5678
短信验证码：[______] [发送验证码]
[取消] [确认启用]
```

### 交互

- 不自动发送短信。
- 用户点击「发送验证码」后进入 60 秒倒计时。
- 验证码为空时确认按钮不可用。
- 验证码错误或过期时停留当前表单。
- 启用成功后展示完成态，不自动回跳原业务操作。

### 异常

- 无可用手机号：展示阻断态，引导绑定手机号。
- 短信发送限流：展示安全服务返回的等待时间。
- 短信服务异常：禁用确认，提示稍后重试。

## W-03 海外账号启用验证器动态码

### 结构

```text
Step 1 扫码或手动输入密钥
-> Step 2 输入验证器 6 位动态码
-> Step 3 启用完成
```

### 交互

- 二维码和手动密钥只用于初始化展示，不进入日志、埋点或截图附件。
- 动态码为空时确认按钮不可用。
- 动态码错误或过期时停留 Step 2，并提示检查设备时间。
- 启用成功后展示完成态，不自动回跳原业务操作。

## W-04 支付密码设置 / 重置合并表单

### 未命中 2FA

```text
安全验证并设置支付密码
当前登录密码：[______]
新支付密码：[______]
确认新支付密码：[______]
[取消] [确认设置]
```

### 命中大陆短信 2FA

```text
安全验证并重置支付密码
当前登录密码：[______]
短信验证码：[______] [发送验证码]
新支付密码：[______]
确认新支付密码：[______]
[取消] [确认重置]
```

### 命中海外 TOTP

```text
安全验证并重置支付密码
当前登录密码：[______]
动态验证码：[______] [无法使用验证器？]
新支付密码：[______]
确认新支付密码：[______]
[取消] [确认重置]
```

### 交互

- 当前登录密码固定必填。
- 邮箱验证码不再作为固定因子。
- 两步验证是否展示由安全服务策略决定。
- 不做登录密码或 2FA 独立通过态。
- 任一安全因子失败时停留表单，清空安全输入和支付密码输入。
- 成功后只展示支付密码设置 / 重置成功。

## W-05 资金确认合并表单

### 大陆短信

```text
资金操作摘要
支付密码：[______]
短信验证码：[______] [发送验证码]
[返回修改] [确认提交]
```

### 海外 TOTP

```text
资金操作摘要
支付密码：[______]
动态验证码：[______] [无法使用验证器？]
[返回修改] [确认提交]
```

### 交互

- 支付密码和 2FA 同屏输入。
- 不拆成支付密码预校验和 2FA 预校验。
- 最终提交同请求校验业务条件、支付密码、2FA 和幂等。
- 返回修改、关闭弹窗或离开页面时清空支付密码和 2FA 输入。
- 支付密码或 2FA 失败时保留业务摘要，清空安全输入。
- 业务规则失败时保留业务摘要并展示业务原因。

## W-06 2FA 阻断态

### 未初始化

```text
当前操作需要两步验证
请先到安全中心启用两步验证后重新发起本次操作。
[去安全中心设置] [返回]
```

### 已锁定

```text
两步验证暂不可用
验证码错误次数过多，预计 24 分钟后可重试。
[查看找回方式] [返回]
```

### 不可用

```text
两步验证通道暂不可用
请稍后重试或联系平台客服。
[联系客服] [返回]
```

## W-07 提交时风险升高补充 2FA

### 场景

预查策略为 `requires2FA=false`，最终提交时重新评估为 `requires2FA=true`。

### 页面

```text
为了保护账户安全，本次操作需要补充两步验证。
资金操作摘要
支付密码：[重新输入]
短信验证码 / 动态验证码：[______]
[返回修改] [确认提交]
```

### 交互

- 不执行资金动作。
- 清空支付密码输入。
- 追加两步验证输入。
- 用户补充后再次提交，后端再次评估并校验。

## 响应式说明

- 桌面端：安全中心左侧为安全项列表，右侧为当前线框状态与说明；业务确认表单居中展示。
- 移动端：单列展示，安全项、提示、表单按顺序堆叠。
- 表单按钮移动端全宽展示，避免窄屏文字拥挤。

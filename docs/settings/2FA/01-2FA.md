# 2FA

## 1. 文档信息

| 项目 | 内容 |
|---|---|
| 文档名称 | 支付密码接入两步验证 PRD |
| 所属目录 | `/2FA` |
| 当前版本 | v1.0 |
| 日期 | 2026-06-16 |
| 阶段 | `prd-generator` |
| 关联能力 | 支付密码、账号设置 / 安全中心、代理商提现、代理商给下级用户充值、安全服务策略编排 |

## 2. 背景与问题陈述

现有支付密码已经作为 `账号设置 / 安全中心` 的资金安全能力，用于提现、给下级用户充值等高敏资金操作。原支付密码 P0 不包含两步验证。随着资金操作安全要求提升，需要在不破坏支付密码独立性的前提下，将支付密码相关资金和安全场景接入两步验证。

本需求将 2FA 定义为独立账号安全能力。2FA 的初始化、状态展示、锁定提示和找回入口统一位于 `账号设置 / 安全中心`；支付密码设置 / 重置、提现和充值只消费安全服务返回的策略、2FA 类型、2FA 状态和校验结果。

中国大陆账号使用手机短信作为 2FA；海外账号使用 TOTP 验证器动态码。环境按照账号所属环境固定判断，不按照本次访问 IP 动态切换。

## 3. 目标

1. 在安全中心提供两步验证的初始化、状态展示、锁定 / 不可用提示和找回入口。
2. 将支付密码设置 / 重置接入安全服务策略，去掉原固定邮箱验证码，使用合并表单完成登录密码、可选 2FA、新支付密码校验。
3. 提现申请 P0 全量要求 `支付密码 + 两步验证`。
4. 给下级用户充值默认要求支付密码，并由安全服务按风险决定是否叠加两步验证。
5. 通过安全服务统一编排策略，避免 M02 / M05 等业务页写死短信、TOTP 或风险规则。
6. 在最终提交时重新评估安全策略，并同请求校验业务条件、支付密码、2FA 和幂等。
7. 明确审计、埋点和敏感信息采集边界，防止支付密码、验证码、TOTP 密钥等敏感信息泄露。

## 4. 范围与非目标

### 4.1 In Scope

- 2FA 独立账号安全能力的产品边界。
- 安全中心两步验证安全项。
- 中国大陆账号手机短信 2FA 初始化。
- 海外账号 TOTP 2FA 初始化。
- 2FA 状态展示：未初始化、已启用、已锁定、不可用。
- 支付密码设置 / 重置合并表单。
- 支付密码设置 / 重置去掉固定邮箱验证码。
- 提现申请全量接入 `支付密码 + 两步验证`。
- 给下级用户充值按风险触发 2FA。
- 资金确认页支付密码和 2FA 同屏输入。
- 两段式安全策略评估：预查策略、提交时重新评估。
- 提交时风险升高后的补充 2FA。
- 2FA 未初始化、锁定、不可用时强阻断。
- 支付密码失败计数和 2FA 失败计数独立。
- 审计、埋点、敏感信息边界和灰度开关。

### 4.2 Out of Scope

| 非目标 | 说明 |
|---|---|
| 在支付密码模块内实现 2FA 初始化 | 2FA 是独立账号安全能力，由安全中心承载 |
| 在业务页写死短信或 TOTP 规则 | 业务页只消费安全服务策略 |
| 关闭 2FA | 属于完整账号安全管理高敏动作，P0 不纳入 |
| 换绑手机号 | 手机号绑定 / 修改是独立账号安全流程 |
| 换绑 TOTP | 属于完整 2FA 管理需求，P0 不纳入 |
| 恢复码生成、展示、下载、吊销、重新生成 | 恢复码是独立敏感凭证体系，本轮不做 |
| 完整 2FA 丢失找回流程 | P0 只提供找回入口或客服入口 |
| 账号环境迁移 | 大陆 / 海外环境按账号所属环境固定，迁移另起需求 |
| 后台安全策略配置页面 | 本轮定义策略能力和消费方式，不建设运营配置后台 |
| 接口字段最终命名和数据库表结构 | PRD 定义能力、枚举和约束，研发按现有规范命名 |

## 5. 角色与权限

| 角色 | 能力 | 说明 |
|---|---|---|
| 中国大陆账号用户 | 初始化手机短信 2FA、使用短信验证码完成高敏操作 | 前提是账号已有可用绑定手机号 |
| 海外账号用户 | 初始化 TOTP、使用验证器动态码完成高敏操作 | P0 不生成恢复码 |
| 代理商用户 | 提现、给下级用户充值、设置 / 重置支付密码 | 高敏步骤按安全服务策略展示验证或阻断 |
| 安全服务 | 策略评估、短信挑战、TOTP 校验、状态和锁定管理、审计 | 是 2FA 策略的唯一来源 |
| 资金服务 | 提现、充值业务校验和资金动作执行 | 最终提交时调用安全服务重新评估并校验 |
| 账号安全服务 | 手机号绑定 / 修改、2FA 找回入口 | 本 PRD 只定义跳转动作，不展开完整流程 |

## 6. 全局业务规则

1. 2FA 是独立账号安全能力，不属于支付密码模块内部能力。
2. 中国大陆账号使用手机短信 2FA，海外账号使用 TOTP 2FA。
3. 账号环境按账号所属环境固定判断，不按本次访问 IP 动态切换。
4. 异常访问地区只作为风险因子，不改变 2FA 类型。
5. 安全服务是 2FA 触发策略、2FA 类型、状态、阻断动作的唯一来源。
6. 前端和业务服务不得自行计算是否需要 2FA。
7. 进入高敏步骤前可预查策略，用于决定页面展示；最终提交必须重新评估策略。
8. 提交时策略升级为需要 2FA 时，不执行资金或安全动作，返回补充 2FA 状态。
9. 策略查询失败时，高敏操作禁用提交，不允许降级绕过支付密码或 2FA。
10. 命中 2FA 但状态为未初始化、锁定或不可用时，P0 强阻断。
11. 2FA 初始化完成后停留在安全中心，不自动回跳提现、充值或支付密码设置 / 重置流程。
12. 支付密码设置 / 重置固定要求当前登录密码，不再固定要求邮箱验证码。
13. 提现申请 P0 全量要求 `支付密码 + 两步验证`。
14. 给下级用户充值默认要求支付密码，2FA 由安全服务按风险触发。
15. 资金页支付密码和 2FA 在同一个确认表单中输入，不拆成多个预校验步骤。
16. 最终提交同请求校验业务条件、支付密码、2FA 和幂等。
17. 支付密码失败计数和 2FA 失败计数独立；任一锁定只阻断需要该因子的操作。
18. 关闭弹窗、返回修改或离开页面时，清空支付密码和 2FA 输入，不写入本地存储。
19. 日志、埋点、审计不得记录支付密码、短信验证码、TOTP 动态码、TOTP secret、二维码内容、手动密钥、恢复码或支付密码哈希。
20. 灰度开关只允许关闭 2FA 触发策略，不允许绕过支付密码。

## 7. 功能需求

### 7.1 M01 安全中心两步验证初始化与状态

#### 7.1.1 安全中心安全项

安全中心需要新增独立的“两步验证”安全项，与“支付密码”安全项并列展示。

| 状态 | 展示内容 | 操作 |
|---|---|---|
| 未初始化 | 两步验证未启用、当前账号将使用的验证方式 | 启用两步验证 |
| 已启用 | 已启用、验证方式、最近启用时间 | 查看找回方式 |
| 已锁定 | 锁定原因、预计可重试时间 | 查看找回方式 / 联系客服 |
| 不可用 | 通道不可用或缺少必要绑定 | 去绑定手机号 / 稍后重试 / 联系客服 |

#### 7.1.2 中国大陆账号手机短信初始化

适用账号环境：`mainland`。

前置条件：

- 账号已有可用绑定手机号。
- 安全服务返回 2FA 类型为 `sms`。

流程：

1. 用户在安全中心点击“启用两步验证”。
2. 页面展示“启用手机短信验证”、脱敏手机号和短信验证码输入框。
3. 用户手动点击“发送验证码”。
4. 安全服务发送短信验证码并返回倒计时、限流或通道状态。
5. 用户输入 6 位短信验证码并点击确认启用。
6. 安全服务校验短信验证码。
7. 校验通过后启用 2FA，安全中心展示“已启用 · 手机短信”。

规则：

- 短信验证码为 6 位数字。
- 验证码有效期默认 10 分钟，最终以安全服务配置为准。
- 重新发送倒计时默认 60 秒，最终以安全服务返回为准。
- 短信不自动发送，必须由用户点击发送。
- 手机号展示为脱敏格式，例如 `+86 138****5678`。
- 验证码错误、过期或限流时停留当前表单。
- 没有可用绑定手机号时，不在 2FA 初始化流程内绑定手机号，展示阻断并引导到独立绑定 / 更新手机号流程。

#### 7.1.3 海外账号 TOTP 初始化

适用账号环境：`overseas`。

流程：

1. 用户在安全中心点击“启用两步验证”。
2. Step 1 展示二维码和手动密钥。
3. 用户使用验证器 App 扫码或手动输入密钥。
4. Step 2 用户输入 6 位动态码。
5. 安全服务校验 TOTP。
6. 校验通过后启用 2FA。
7. Step 3 展示启用成功，安全中心展示“已启用 · 验证器动态码”。

规则：

- TOTP 动态码为 6 位数字。
- 服务端允许前后 1 个时间窗口的轻微漂移。
- 动态码错误或过期时停留 Step 2，并提示用户检查设备时间。
- TOTP 输入旁展示“无法使用验证器？”入口，指向独立账号安全找回或客服入口。
- P0 不生成、不展示、不下载、不管理恢复码。
- 二维码、手动密钥、TOTP secret、动态码不得进入日志、埋点、截图附件或普通审计字段。

### 7.2 M02 支付密码设置 / 重置接入 2FA

#### 7.2.1 入口与策略预查

用户从安全中心进入设置或重置支付密码时，前端调用安全服务预查策略，场景分别为：

- `set_payment_password`：设置支付密码。
- `reset_payment_password`：重置支付密码。

预查结果用于决定是否展示 2FA 输入或阻断状态。

#### 7.2.2 合并表单

支付密码设置 / 重置使用同一个合并表单模式。

未命中 2FA：

```text
当前登录密码
新支付密码
确认新支付密码
确认设置 / 确认重置
```

命中大陆短信 2FA：

```text
当前登录密码
短信验证码 + 发送验证码
新支付密码
确认新支付密码
确认设置 / 确认重置
```

命中海外 TOTP：

```text
当前登录密码
动态验证码 + 无法使用验证器入口
新支付密码
确认新支付密码
确认设置 / 确认重置
```

规则：

- 当前登录密码固定必填。
- 邮箱验证码不再作为固定验证因子。
- 两步验证是否展示由安全服务策略决定。
- 不做登录密码或 2FA 的独立通过态。
- 点击确认后，后端同一次校验登录密码、2FA、支付密码规则和幂等。
- 任一安全因子失败时，不保存新支付密码。
- 失败后停留当前表单，清空当前登录密码、2FA 输入、新支付密码、确认新支付密码。
- 成功后刷新支付密码状态，展示设置 / 重置成功。
- 成功后不自动跳回资金操作。

#### 7.2.3 支付密码规则

支付密码规则沿用原支付密码能力：

- 6 位数字 PIN。
- 新支付密码和确认新支付密码必须一致。
- 不允许空值。
- 支付密码哈希、失败计数、锁定规则由支付密码服务管理。
- 支付密码设置 / 重置中的 2FA 失败计数由安全服务管理。

### 7.3 M03 资金业务页支付密码 + 2FA 消费

#### 7.3.1 提现申请

提现申请 P0 全量要求 `支付密码 + 两步验证`。

流程：

1. 用户填写提现金额、处理方式和收款对象。
2. 点击提交或下一步时，前端以 `withdraw` 场景预查安全策略。
3. 若 2FA 状态为 `enabled`，展示资金操作摘要、支付密码输入和短信 / TOTP 输入。
4. 若 2FA 状态为 `uninitialized`、`locked` 或 `unavailable`，展示阻断态。
5. 用户输入支付密码和 2FA 后提交。
6. 资金服务在后端最终提交时调用安全服务重新评估并校验。
7. 校验通过后，再校验账单、窗口、金额、幂等等业务条件。
8. 全部通过后创建提现申请并冻结金额。

阻断规则：

- 未初始化 2FA 时，不创建提现申请，不冻结金额，不保存草稿。
- 2FA 锁定或不可用时，不创建提现申请，不冻结金额。
- 安全服务或策略查询异常时，禁用提交并提示稍后重试。

#### 7.3.2 给下级用户充值

给下级用户充值默认要求支付密码，是否叠加 2FA 由安全服务按风险触发。

风险因子包括：

- 单笔金额超过安全阈值。
- 短时间内多次充值。
- 新设备或可信设备缺失。
- 异常登录地区。
- 账号风险命中。
- 支付密码或 2FA 近期失败次数较多。

流程：

1. 代理商填写充值金额和下级用户信息。
2. 点击下一步或确认时，以 `agent_user_recharge` 场景预查安全策略。
3. 若 `requires2FA=false`，展示支付密码确认。
4. 若 `requires2FA=true` 且 2FA 可挑战，展示支付密码和 2FA 合并确认。
5. 若 2FA 不可挑战，展示阻断态。
6. 最终提交时重新评估策略并校验代理身份、用户归属、余额、风控、支付密码、2FA 和幂等。
7. 全部通过后执行代理钱包扣减和用户钱包增加。

#### 7.3.3 资金确认合并表单

资金确认表单统一结构：

```text
资金操作摘要
支付密码
两步验证输入（命中时）
返回修改
确认提交
```

规则：

- 支付密码和 2FA 同屏输入。
- 不拆成支付密码预校验、2FA 预校验、资金提交三个独立动作。
- 返回修改、关闭弹窗或离开页面时，清空支付密码和 2FA 输入。
- 支付密码或 2FA 失败时保留业务摘要，清空安全输入。
- 业务规则失败时保留业务摘要并展示业务失败原因。

#### 7.3.4 提交时风险升高

当预查策略返回 `requires2FA=false`，但最终提交重新评估后返回 `requires2FA=true`：

- 后端不执行资金或安全动作。
- 前端展示同一个确认表单，并追加两步验证输入。
- 前端清空支付密码输入。
- 用户需要重新输入支付密码和 2FA 后再次提交。
- 再次提交时后端仍需重新评估策略。

### 7.4 M04 安全策略服务、审计与灰度

#### 7.4.1 安全策略服务职责

安全服务负责：

- 判断账号所属环境。
- 判断 2FA 类型。
- 判断本次场景是否需要支付密码。
- 判断本次场景是否需要 2FA。
- 返回 2FA 状态、阻断原因和恢复动作。
- 发起大陆短信验证码挑战。
- 校验短信验证码或 TOTP 动态码。
- 管理 2FA 失败计数、锁定和解锁。
- 写入安全审计日志。

#### 7.4.2 两段式策略评估

| 阶段 | 触发时机 | 目的 | 结果 |
|---|---|---|---|
| 预查策略 | 进入高敏步骤前 | 决定页面展示输入、阻断或重试 | 只用于展示，不代表最终允许执行 |
| 提交时重新评估 | 用户点击最终提交后 | 决定是否允许执行业务动作 | 后端最终裁决 |

#### 7.4.3 灰度与回滚

- 灰度可按账号环境、用户分组、业务场景开启。
- 提现全量 2FA 的灰度节奏由安全和资金团队共同控制。
- 灰度开关只允许关闭 2FA 触发策略，不允许绕过支付密码。
- 所有策略关闭、阈值调整和异常回滚必须写入审计。

## 8. 页面与字段说明

### 8.1 安全中心两步验证安全项

| 字段 / 元素 | 类型 | 必填 | 说明 |
|---|---:|---:|---|
| 两步验证状态 | 文本 / 标签 | 是 | 未初始化、已启用、已锁定、不可用 |
| 验证方式 | 文本 | 否 | 手机短信或验证器动态码，未启用时展示即将使用的方式 |
| 最近启用时间 | 文本 | 否 | 已启用态展示 |
| 锁定原因 | 文本 | 否 | 已锁定态展示 |
| 剩余可重试时间 | 文本 | 否 | 已锁定态展示 |
| 恢复动作按钮 | 按钮 | 否 | 启用、去绑定手机号、查看找回方式、联系客服、稍后重试 |

### 8.2 大陆短信初始化表单

| 字段 / 元素 | 类型 | 必填 | 校验 |
|---|---:|---:|---|
| 脱敏手机号 | 文本 | 是 | 只展示脱敏信息 |
| 短信验证码 | 输入框 | 是 | 6 位数字 |
| 发送验证码 | 按钮 | 是 | 点击后进入倒计时 |
| 确认启用 | 按钮 | 是 | 验证码为空时不可提交 |

## 9. 流程与状态机

### 9.1 安全策略评估主流程

```text
进入高敏步骤
-> 收集场景和风险上下文
-> 预查安全策略
-> 根据策略展示支付密码、2FA 输入或阻断
-> 用户最终提交
-> 后端重新评估策略
-> 同请求校验业务条件、支付密码、2FA 和幂等
-> 执行动作或返回失败原因
```

### 9.2 2FA 状态机

```text
未初始化
-> 初始化中
-> 已启用
-> 验证中
-> 已启用 / 已锁定 / 不可用
-> 找回或恢复入口
-> 已启用或重新初始化
```

状态说明：

- `uninitialized`：未初始化两步验证。
- `enabled`：已启用且可正常验证。
- `locked`：因错误次数或风控暂时锁定。
- `unavailable`：通道不可用或缺少必要绑定。

完整 Mermaid 图集见 `2FA/2026-06-16-payment-password-2fa-mermaid.md`。

## 10. 数据与接口能力

本 PRD 定义能力与字段语义，不锁定最终接口路径和数据库表结构。

### 10.1 策略预查能力

输入能力：

| 字段 | 说明 |
|---|---|
| `scene` | 场景枚举，如提现、充值、设置支付密码、重置支付密码 |
| `riskContext` | 金额、币种、设备、地区、频次、账号风险等上下文 |
| `bizType` | 业务类型，用于细分资金或安全动作 |

输出能力：

| 字段 | 说明 |
|---|---|
| `policyId` / `policyToken` | 策略标识，用于串联预查、短信发送、最终提交和审计 |
| `scene` | 本次策略所属场景 |
| `env` | 账号所属环境 |
| `requiresPaymentPassword` | 是否需要支付密码 |
| `requires2FA` | 是否需要两步验证 |
| `twoFactorType` | 两步验证类型 |
| `twoFactorStatus` | 两步验证状态 |
| `blockReason` | 阻断原因 |
| `recoveryAction` | 可执行恢复动作 |

### 10.2 短信挑战能力

用于大陆账号 2FA 初始化和高敏操作短信验证。

能力要求：

- 根据 `policyId` / `policyToken` 发起短信验证码。
- 返回发送成功、倒计时、限流、通道异常等状态。
- 不返回明文验证码。
- 支持验证码有效期、重发倒计时、发送限流和错误次数控制。

### 10.3 TOTP 初始化与校验能力

能力要求：

- 为海外账号生成 TOTP 初始化二维码和手动密钥。
- 校验用户输入的 6 位 TOTP 动态码。
- 支持前后 1 个时间窗口漂移。
- 管理 TOTP 错误次数、锁定和解锁。
- 不在普通日志、埋点、截图或审计字段中记录 secret、二维码内容、手动密钥和动态码。

### 10.4 最终提交安全校验能力

最终提交时，业务服务必须在后端重新评估策略，并完成以下校验：

- 业务条件。
- 幂等。
- 支付密码。
- 2FA 验证码或 TOTP。
- 账号、角色、资金归属和风控状态。

## 13. 安全审计

必须写入安全审计的事件：

- 2FA 初始化开始、成功、失败。
- 短信验证码发送成功、发送失败、限流。
- TOTP 校验成功、失败、锁定。
- 支付密码设置 / 重置安全校验成功、失败。
- 提现最终提交安全校验成功、失败。
- 充值最终提交安全校验成功、失败。
- 策略查询失败。
- 提交时策略升级。
- 2FA 未初始化、锁定、不可用阻断。
- 灰度开关变更、策略关闭、阈值调整和异常回滚。

审计字段建议包含：

- 用户 ID、账号环境、场景、策略标识、设备标识、IP 区域、业务单号、结果、失败原因、时间。

审计字段禁止包含：

- 支付密码、当前登录密码、短信验证码、TOTP 动态码、TOTP secret、二维码内容、手动密钥、恢复码、支付密码哈希。

## 14. 验收标准

### 14.1 安全中心

- 中国大陆账号可在安全中心启用手机短信 2FA。
- 中国大陆账号没有可用手机号时，展示阻断并引导绑定 / 更新手机号。
- 海外账号可在安全中心启用 TOTP。
- 海外账号 TOTP 初始化包含二维码、手动密钥、6 位动态码校验和完成态。
- 2FA 初始化完成后停留安全中心，不自动回跳原操作。
- 已启用、已锁定、不可用状态展示正确。

### 14.2 支付密码设置 / 重置

- 设置 / 重置支付密码不再固定要求邮箱验证码。
- 当前登录密码固定必填。
- 命中 2FA 时，按账号环境展示短信或 TOTP 输入。
- 未命中 2FA 时，只展示当前登录密码、新支付密码、确认新支付密码。
- 任一安全因子失败时，不保存新支付密码，并清空表单安全输入。
- 提交时策略升级为需要 2FA 时，不保存新支付密码，返回补充 2FA。

### 14.3 提现

- 提现申请 P0 全量展示支付密码和两步验证。
- 未初始化 2FA 时，不创建提现申请，不冻结金额。
- 2FA 锁定或不可用时，不创建提现申请，不冻结金额。
- 最终提交时重新评估策略。
- 全部安全和业务校验通过后，才创建提现申请并冻结金额。

### 14.4 给下级用户充值

- 给下级用户充值默认要求支付密码。
- 安全服务按风险返回是否需要 2FA。
- 命中 2FA 时，支付密码和 2FA 同屏输入。
- 提交时风险升高时，不执行转账，要求补充 2FA。
- 全部安全和业务校验通过后，才执行代理钱包扣减和用户钱包增加。

### 14.5 安全与数据

- 策略查询失败时，高敏操作禁用提交。
- 支付密码失败计数和 2FA 失败计数独立。
- 敏感字段不进入日志、埋点、审计或截图附件。
- 灰度关闭 2FA 触发策略时，不影响支付密码要求。

## 15. 上线计划

1. 联调安全服务策略预查、短信挑战、TOTP 初始化、最终提交校验能力。
2. 安全中心上线两步验证安全项和初始化流程。
3. 支付密码设置 / 重置切换为合并表单，并移除固定邮箱验证码。
4. 资金页接入策略预查和合并确认表单。
5. 提现按灰度开启全量 `支付密码 + 两步验证`。
6. 给下级用户充值开启按风险触发 2FA。
7. 上线埋点、审计和告警。
8. 灰度观察策略查询失败率、短信发送失败率、TOTP 校验失败率、阻断率、资金提交成功率。
9. 达到安全和业务指标后全量发布。

## 17. 依赖与风险

| 依赖方 | 依赖内容 | 风险 | 应对 |
|---|---|---|---|
| 账号设置 / 安全中心 | 两步验证安全项、支付密码入口、手机号绑定入口 | 安全中心信息架构影响入口落地 | PRD 明确页面结构和入口动作 |
| 安全服务 | 策略评估、短信挑战、TOTP 校验、状态、锁定、审计 | 策略不统一会导致业务页重复实现 | 明确安全服务是唯一策略来源 |
| 短信服务 | 大陆短信验证码发送、限流和错误返回 | 通道异常会影响大陆账号高敏操作 | 高敏操作默认禁用，提示重试或客服 |
| TOTP 能力 | TOTP secret、二维码、动态码校验、时间窗口 | 密钥泄露或日志误采集风险高 | 明确 secret / 二维码 / 动态码不得进入日志与埋点 |
| 资金服务 | 提现、充值最终提交同请求校验 | 预查和提交分离可能产生竞态 | 提交时重新评估策略 |
| 代理商 M02 / M05 | 充值、提现页面文案和确认表单适配 | 旧文案“支付密码 / 二次校验”容易混淆 | 统一使用“支付密码”和“两步验证” |
| 数据平台 | 埋点和审计日志 | 敏感信息泄露风险 | 明确禁止采集字段和审计边界 |
